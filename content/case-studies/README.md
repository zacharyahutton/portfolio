# Case studies

Long-form case study content lives in `content/case-studies/*.ts` and is attached to projects at build time via `content/projects/index.ts`.

Routes are generated from projects that include a case study:

- `/projects/weroi`
- `/projects/pntcog`
- `/projects/studysync-api`
- `/projects/webhook-relay-api`
- `/projects/openapi-devkit`
- `/projects/phone-store-api`

Screenshots are stored in `public/case-studies/`.

Each case study includes: problem, solution, architecture sections, key decisions, optional metrics, and stack pills.
